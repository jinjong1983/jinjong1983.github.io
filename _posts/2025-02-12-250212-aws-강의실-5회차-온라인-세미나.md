aws mla 자격증 취득할때만 공부해고 실무에선 사용해보지 않았던 서비스.. 프로젝트 진행과정 설계 구현 등에대해서 전체적으로 사례를 공유받을 수 있었다. 나도 하나 해볼까 싶긴한데 아직까지는 챗gpt api 따서 쓰는거 이상의 직접 학습을 돌리거나 하는 부분은 좀 접근의 허들이 있다... ollma 와의 비교분석 비용 등.

## Amazon Aurora 변경 내역을 스트림으로 받아 분석처리는(오타일듯) Serverless 아키텍처
관심이 있던 부분이였다. aurora에 발생하는 이벤트를 기반으로 하는 일종의 cdc에 관련 된 내용. 
- aurora activity streams 사용하는곳은 kinesis, kms
- 지원 인스턴스 타입이 제한적임(db 생성부터 실제 구현영상)
- db 생성하고 활동스트림 구성 옵션을 켜야함. 
- 구독할때 kms 를 디코딩 해야하는데 이부분 참조할 코드들이 많지않다 ( iam 으로 접근하면 디코딩은 aws가 담당해주는 줄 알았는데...)
- 데이터베이스 리소스id = 클러스터id
- 오로라의 키네시스 스트림을 붙여서 쉽게 연동가능한데... 이건 비용계산 살짝 봐야할듯
- 실제로 mysql 에 ddl dml 날리는걸 보면서 실습과정 종료
- 와치보다 빠르다. 와치는 이벤트기반은 아니니까..
